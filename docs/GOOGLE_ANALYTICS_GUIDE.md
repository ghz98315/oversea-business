## 主要报告说明

### 1. 实时报告 📍

**路径**: 报告 → 实时

**功能**: 查看当前正在访问网站的用户

**主要指标**:
- **用户数**: 当前在线人数
- **过去 30 分钟的用户**: 最近半小时的访问量
- **事件数**: 正在发生的交互次数
- **热门页面**: 用户正在浏览的页面

**使用场景**:
- ✅ 发布新内容后立即查看反应
- ✅ 社交媒体推广后追踪效果
- ✅ 验证 GA 代码是否正常工作
- ✅ 监控突发流量

**查看方式**:
```
报告 → 实时 → 查看实时数据
```

---

### 2. 流量获取报告 🚀

**路径**: 报告 → 生命周期 → 客户开发 → 流量获取

**功能**: 了解用户从哪里来

**主要渠道**:

| 渠道 | 说明 | 优化建议 |
|------|------|----------|
| **Organic Search** | 搜索引擎自然流量（Google、Bing等） | SEO 优化最重要的指标 |
| **Direct** | 直接访问（输入网址、书签） | 品牌认知度的体现 |
| **Referral** | 从其他网站链接点击 | 外链和合作效果 |
| **Social** | 社交媒体（LinkedIn、Facebook等） | 社媒营销效果 |
| **Paid Search** | 付费搜索广告（Google Ads） | 广告投放效果 |
| **Email** | 邮件营销 | 邮件列表质量 |

**关键指标**:
- **用户数**: 不同渠道带来的访客数量
- **新用户**: 首次访问的用户
- **互动率**: 用户参与度（滚动、点击、停留）
- **转化次数**: 完成目标行为的次数

**使用场景**:
- ✅ 评估哪个渠道最有效
- ✅ 决定营销预算分配
- ✅ 识别增长机会

**查看方式**:
```
报告 → 生命周期 → 客户开发 → 流量获取
点击 "会话默认渠道分组" 查看渠道详情
```

### 3. 互动度报告 💡

**路径**: 报告 → 生命周期 → 互动度

#### 3.1 页面和屏幕

**功能**: 查看哪些页面最受欢迎

**主要指标**:
- **浏览量**: 页面被查看的次数
- **用户数**: 访问该页面的独立用户数
- **平均互动时长**: 用户在页面停留的时间
- **跳出率**: 只浏览一页就离开的比例

**重点关注的页面**:
1. `/` - 首页（第一印象）
2. `/blog` - 博客列表（内容营销效果）
3. `/supplier-match` - 供应商匹配（核心转化页）
4. `/suppliers/[id]` - 供应商详情（用户兴趣）
5. `/quote` - 询盘页面（转化页面）

**优化建议**:
- 浏览量高但互动时长短 → 内容不够吸引人
- 跳出率高 → 页面内容与期望不符
- 互动时长长 → 内容有价值或太复杂

#### 3.2 事件

**功能**: 查看用户交互行为

**自动收集的事件**:
- `page_view` - 页面浏览
- `session_start` - 会话开始
- `first_visit` - 首次访问
- `scroll` - 页面滚动
- `click` - 点击

**自定义事件**（我们已添加）:
- `form_submit` - 表单提交
- `quote_request` - 询盘请求 ⭐ 最重要
- `sample_request` - 样品申请
- `email_capture` - 邮件订阅
- `button_click` - 按钮点击
- `supplier_view` - 供应商查看
- `blog_read` - 博客阅读

**查看方式**:
```
报告 → 生命周期 → 互动度 → 事件
点击事件名称查看详细数据
```

## 关键指标解读

### 核心指标

#### 1. 用户 vs 新用户
- **用户**: 访问网站的总人数（去重）
- **新用户**: 首次访问的用户数
- **回访用户**: 用户数 - 新用户

**健康指标**:
- 新用户占比 60-80%: 增长良好
- 回访率 20-40%: 内容有粘性

#### 2. 会话（Sessions）
- 一次访问算一个会话
- 30 分钟无活动后结束
- 或午夜 12 点自动结束

#### 3. 互动率（Engagement Rate）
- 停留超过 10 秒、或有 2 次以上浏览、或触发转化事件
- 替代了旧版的"跳出率"

**健康指标**:
- B2B 网站: 40-60% 算良好
- 低于 30%: 流量质量差或内容不匹配

#### 4. 平均互动时长
- 用户在网站停留的平均时间
- 只计算有互动的会话

**参考值**:
- B2B 服务网站: 2-4 分钟正常
- 博客文章: 5-8 分钟为佳

## 常用场景分析

### 场景 1：评估营销活动效果

**目标**: 知道广告/推广是否有效

**步骤**:
1. 进入 **流量获取** 报告
2. 选择时间范围（活动前后对比）
3. 查看各渠道的:
   - 用户数增长
   - 互动率
   - 转化次数

**判断标准**:
- ✅ 用户数增加 + 互动率正常 = 流量质量好
- ⚠️ 用户数增加 + 互动率低 = 流量质量差

---

### 场景 2：优化转化漏斗

**目标**: 找出用户流失点

**步骤**:
1. 探索 → 路径探索
2. 选择起始点（如首页）
3. 查看用户路径:
   ```
   首页 → 服务页 → 供应商页 → 询盘页
   ```
4. 找出流失最多的环节

**优化方向**:
- 流失多 → 优化该页面内容/设计
- 转化低 → 优化 CTA 按钮

---

### 场景 3：内容效果分析

**目标**: 了解哪些内容最有效

**步骤**:
1. 互动度 → 页面和屏幕
2. 筛选博客页面（`/blog/`）
3. 按互动时长排序

**指标解读**:
- 高浏览量 + 高互动时长 = 热门优质内容
- 高浏览量 + 低互动时长 = 标题党或内容不符
- 低浏览量 + 高互动时长 = 小众但精准

---

### 场景 4：地区市场分析

**目标**: 了解不同地区用户行为

**步骤**:
1. 用户 → 用户属性 → 国家/地区
2. 对比不同地区的:
   - 用户数
   - 互动率
   - 转化率

**应用**:
- 识别主要市场
- 针对性内容优化
- 多语言版本规划

## 转化目标设置

### 什么是转化？
用户完成你期望的重要行为，如：
- 提交询盘表单
- 订阅邮件
- 下载资源

### 设置关键事件（转化）

**步骤**:
```
1. 管理 → 数据显示 → 事件
2. 找到要标记为转化的事件（如 quote_request）
3. 点击右侧开关"标记为转化"
```

**建议标记为转化的事件**:
- ✅ `quote_request` - 询盘（最重要）
- ✅ `sample_request` - 样品申请
- ✅ `email_capture` - 邮件订阅
- ✅ `form_submit` - 表单提交

### 查看转化数据

**路径**: 报告 → 生命周期 → 客户开发 → 转化次数

**分析**:
- 转化总数
- 转化率（转化次数/用户数）
- 各渠道转化效果对比

## 常见问题

### Q1: 为什么实时报告没有数据？

**可能原因**:
1. GA 代码未正确安装
2. 浏览器安装了广告拦截器
3. 刚部署，需要等待几分钟

**检查方法**:
```
1. 打开网站
2. 按 F12 打开开发者工具
3. Console 标签查看是否有错误
4. Network 标签搜索 "google-analytics"
```

---

### Q2: 数据延迟多久？

- **实时报告**: 即时（几秒钟）
- **标准报告**: 24-48 小时完全处理
- **事件数据**: 通常几小时内可见

---

### Q3: 如何排除内部流量？

**步骤**:
```
1. 管理 → 数据流 → 网络数据流
2. 配置标记设置
3. 创建规则排除你的 IP 地址
```

---

### Q4: 转化率多少算正常？

**B2B 网站参考值**:
- **整体转化率**: 2-5% 良好
- **询盘转化率**: 1-3% 正常
- **邮件订阅率**: 3-5% 良好

**注意**: 行业差异很大，重点是持续提升

---

### Q5: 如何导出数据？

**方法 1**: 直接导出
```
报告右上角 → 分享报告 → 下载文件（PDF/CSV）
```

**方法 2**: Google Sheets 集成
```
报告右上角 → 分享报告 → 在 Google 表格中打开
```

5. **事件报告 - quote_request** (2 分钟)
   - 询盘详细数据
   - 来源分析

**总耗时**: 15 分钟

## 🎯 优化建议

### 基于数据优化网站

1. **流量质量差**（互动率低）
   - 优化 SEO 关键词，吸引精准流量
   - 改善着陆页内容

2. **转化率低**
   - 简化询盘表单
   - 添加更多信任元素
   - 优化 CTA 按钮

3. **页面跳出率高**
   - 检查页面加载速度
   - 改善内容相关性
   - 添加内部链接

4. **特定页面流失**
   - 用户体验测试
   - A/B 测试不同版本
